# Modeling Human Activity States Using Hidden Markov Models

Individual assignment — Machine Learning, July 2026.

## What this project does

Classifies four smartphone-sensor activities — **still**, **standing**, **walking**, **jumping** — using one Gaussian HMM per activity class (one-vs-all classification). Raw accelerometer and gyroscope data is collected from a phone, segmented into overlapping windows, and converted into a 120-dimensional multi-scale feature vector. Each activity model is scored with the Viterbi algorithm and the highest log-likelihood wins.

**Results:** 83% accuracy on training data, 88% on held-out unseen recordings.

## Data format

Each CSV has columns: `time, ax, ay, az, gx, gy, gz`  
- `ax, ay, az` — accelerometer (m/s²)  
- `gx, gy, gz` — gyroscope (rad/s)  
- Sampling rates: 100 Hz (first 5 sessions) or 50 Hz (sessions 6–10 + unseen). The notebook auto-detects and resamples to 50 Hz.

## Key design decisions

| Decision | Choice | Why |
|---|---|---|
| Window length | 2 s (100 samples) | Captures ≥1 full walking stride cycle |
| Overlap | 50% | Increases sample count without losing temporal continuity |
| Multi-scale features | 2 s + 1 s sub-window concatenated | Captures both long-term rhythm and short-term statistics |
| Covariance type | Diagonal | ~150 windows per class — full covariance is over-parameterized |
| Hidden states | still=2, others=4 | Still's flat signal causes degenerate transitions with >2 states |
| Restarts | 5 random seeds, keep best | Avoids poor EM local optima |
